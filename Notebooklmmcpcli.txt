NotebookLM Usage Guide (CLI & MCP)
Google NotebookLM を CLI (コマンドライン) および AI エージェント (MCP) から利用するためのガイドです。

1. 認証 (Authentication)
セッションは約20分で期限切れになります。エラーが出た場合は再ログインしてください。

bash
nlm login          # ブラウザを開いてログイン
nlm auth status    # 現在のログイン状態を確認
2. CLI の基本操作
nlm コマンドには、「名詞（リソース）優先」と「動詞（アクション）優先」の2種類のスタイルがあります。

ノートブック操作
nlm notebook list : ノートブック一覧を表示
nlm notebook create "タイトル" : 新規作成
nlm notebook query <ID> "質問" : ソースに基づいて質問に答える
ソース操作
nlm source add <ノートID> --url "https://..." : URLを追加
nlm source add <ノートID> --file "path/to/file.pdf" : ローカルファイルをアップロード
nlm source list <ノートID> : ソース一覧を表示
IDエイリアス (推奨)
長いIDの代わりに短い名前を使えます。

bash
nlm alias set shogi abc123-uuid...
nlm notebook get shogi  # エイリアスで指定可能
3. AI エージェントでの利用 (MCP)
Antigravity, codexCLI, GeminiCLI などの AI に直接ノートブックの内容を問い合わせることができます。

活用例
「このノートブック shogi の内容を要約して」
「shogi のソースにある最新のトレンドについて解説して」
「URL https://... を shogi に追加して」
4. コンテンツ生成 (Studio)
ノートブックを基に様々な形式を自動生成できます。

nlm audio create <ID> --confirm : 音声解説 (Podcast風) を作成
nlm video create <ID> --confirm : 解説動画を生成 (Whiteboard形式など)
nlm report create <ID> --confirm : 学習ガイドやブログ記事を生成
nlm quiz create <ID> --confirm : クイズを生成
生成されたファイルは nlm download でローカルに保存できます：

bash
nlm download audio <ID> --output my_podcast.mp3
5. 高度なリサーチ (Research Mode)
WebやGoogleドライブから情報を自動的に収集・統合します。

nlm research start "Agentic AI 2026" --notebook-id <ID> --mode deep
nlm research status <ID> (完了まで待機)
nlm research import <ID> <TaskID> (収集したソースをノートに取り込む)
TIP

詳しいヘルプは nlm --help または nlm --ai で確認できます。
