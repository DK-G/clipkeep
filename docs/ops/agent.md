# プロジェクト管理エージェント: ClipKeep

## 1. プロジェクト概要
- プロジェクト名: ClipKeep
- カテゴリ: web
- 目的: SNSメディア抽出導線と問題解決ページを統合し、10言語対応の実用的な取得体験を提供する。アラビア語のRTL品質は引き続き重点管理とする。
- 非目的: 全SNS同時対応、高度な編集機能、重い運用自動化の初期実装。

## 2. 主要ドキュメント
- `plan.md`: マスター仕様（変更提案のみ。無断改変禁止）
- `RoadMap.md`: Phase / Epic 管理
- `task.md`: 実行タスク管理
- `diff.md`: 変更報告（毎作業で更新）
- `review.md`: Geminiレビュー記録
- `Changelog.md`: `diff.md` の履歴要約
- `memo.md`: 調査ログ・設計メモ

## 3. エージェント役割
- 開発エージェント (Codex): `task.md` に沿って実装・検証し、`diff.md` と `Changelog.md` を更新する。
- レビューエージェント (Gemini): 変更内容を評価し `review.md` に判定を記録する。

## 4. Codex運用ルール
1. `task.md` の In Progress から着手する。
2. 実装前に影響範囲を確認し、MVPスコープ外は実装しない。
3. 実装後は最低限の検証手順を実行し、結果を `diff.md` に書く。
4. `diff.md` を更新したら、同内容の要約を `Changelog.md` 先頭へ追記する。
5. 作業完了後に `task.md` の状態を更新する。

## 5. 禁止事項
- `plan.md` の直接変更（必要時は提案のみ）。
- `Gemini.md` / `review.md` のレビュー結果セクション改ざん。
- MVP境界を越える実装の先行投入。

## 6. 不具合対応
- 不具合発見時は、先に `diff.md` に事実（再現手順・期待/実際）を記録する。
- 原因断定せず、必要情報を揃えてレビューへ回す。
- 設計変更級は `RoadMap.md` の Design Notes に昇格する。
