# 週次レビュー運用（KPI / 障害 / SEO）

## 目的
- 数値監視と改善タスク化を同一サイクルで回し、運用品質を固定化する。

## 実施タイミング
- 毎週1回（推奨: 月曜）
- 所要: 30-45分

## 参加ロール
- リリース責任者
- 障害一次対応者
- SEO/コンテンツ担当

## レビュー項目
1. KPI
- `extract_submit`
- `extract_completed`
- Solution CTR
- 国/言語別CVR

2. 障害
- 5xx率
- 429率
- degraded発動件数

3. SEO
- Index状況
- 主要ページ順位変動
- canonical / hreflangエラー

## 会議アウトプット
- `今週の課題Top3`
- `次週アクション`（担当者 / 期限 / 成果物）
- `しきい値変更有無`（Rate limit / WAF）
