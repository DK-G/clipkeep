# AdSense BAN対策 運用ガイド（ClipKeep）

## 目的
- AdSense審査/運用での停止・BANリスクを最小化する。
- 2024-2025年の最新ポリシー（AIコンテンツ、無効なトラフィック対策）に対応する。

## 禁止・注意事項
- **著作権侵害の回避**: 権利物の直接配布や侵害を助長する表現を徹底的に排除。
- **誤解を招くCTAの禁止**: 広告を「ダウンロード」や「再生」ボタンに見せる配置を厳禁。
- **過剰広告の禁止**: コンテンツ量に対して広告が支配的（ページ面積の半分以上など）にならないよう調整。
- **報酬誘導の禁止**: クリックを条件に特典を与える、または「サポートのためにクリックしてください」等の文言を禁止。
- **プライベート通信への配置禁止**: (2024年7月追加) DM、個別チャット、ビデオチャット画面内への広告配置は禁止。

## ページ品質基準 (E-E-A-T)
- **AIコンテンツの品質管理**: 
  - AI生成ツールをそのまま使わず、必ず人間が編集・ファクトチェックを行う。
  - 読者にとって「役立つ（Helpful Content）」独自視点や追加情報を加える。
  - 量産を目的とした低品質な自動生成ページは公開しない。
- **法務ページの完備**: Terms, Privacy Policy, Cookies, DMCA ページをフッターに明示。
- **透明性**: 運営者情報と問い合わせフォームを設置。

## 広告実装ルール（技術的対策）
- **誤クリックの徹底防止**:
  - 広告とボタン/リンクの間には **最小30px以上** の余白を設ける。
  - モバイル版では誤タップを防ぐため、さらに十分な間隔を確保。
- **ラベリング**: 広告枠の上部には必ず「スポンサーリンク」または「広告」と明記する。
- **CLS (Layout Shift) 対策**: 広告読み込み時にコンテンツが動かないよう、事前に広告領域の `min-height` を確保する。
- **Above the fold**: ファーストビュー（スクロール前）の広告は1枠までとし、コンテンツが最初に見える状態を維持。

## トラフィック管理と監視
- **無効なトラフィック (IVT) 対策**:
  - **自己クリック厳禁**: 開発・テスト中でも自分の広告は絶対にクリックしない（Google Publisher Toolbar 等を活用）。
  - **不審なトラフィックの遮断**: Cloudflare 等の WAF を導入し、Bot Fight Mode を有効化。データセンター（AWS/Azure/GCP等）からのBotアクセスを制限。
  - **レート制限**: 悪意のある連続リロードによるインプレッション稼ぎを防ぐため、IP単位のレート制限を実装。
- **GA4監視項目**:
  - 特定のIPからのクリック集中がないか。
  - 異常なクリック率（CTR）のスパイク。
  - 参照元（Referral）に不審なドメイン（トラフィック購入、リンク交換サイト等）がないか。

## インシデント対応
1. **ポリシー違反の警告**: AdSenseポリシーセンターを毎日確認し、警告があれば即座に対応。
2. **広告配信の停止**: 問題が特定されるまで、該当ページまたはサイト全体の広告配信をプログラム側で一時停止。
3. **修正と再審査**: 修正内容（配置変更、コンテンツ削除等）を記録し、再審査を申請。
4. **ログの保存**: 無効なトラフィックの証拠（IP、タイムスタンプ）を保存し、不服申し立ての際に提出できるよう準備。
