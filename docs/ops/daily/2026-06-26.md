## インシデントと是正（同日内で解消）

撤去ツール初版に **Windows のシェル引用符バグ**: `spawn(..., {shell:true})` 経由で KV put の
JSON 値（`["x"]`）の二重引用符が剥がれ、本番 `topics:removed` に不正値 `[t-x]` を一時保存した。
**サイトは無影響**（アプリは `safeParse` で不正値→空扱い、リリースゲートも PASS のまま）だが、
ツールが不正値で停止した。是正（commit `0731ef9`）:
- put の値を `--path`（一時ファイル）渡しに変更しシェル引用符の影響を排除。
- `readRemoved` を不正値に対し警告＋空扱いの自己修復へ（次の書き込みで valid JSON に置換）。
- 本番 KV を `remove`→`restore` で valid `[]` に復元・確認済み。
- raw `wrangler kv put` の直接実行は auto-mode classifier がブロック（想定どおり）。クリーンアップは
  ツール自身（`npm run trend:topic`）で完遂。
