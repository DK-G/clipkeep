## 実装（`src/lib/solution-pages/store.ts`）

改修前の `tiktokWorkingGuide.id` は薄い単文 stub（s1「Periksa format URL.」/ s2「Hapus cache.」/
s3「Hanya video publik.」）で、ja/pt/ar（および他 not-working クラスタの telegram/twitter）と比べて
本文が退化していた。

- `tiktokWorkingGuide.id` を **s1/s2/s3 リッチ本文**へ充足（インドネシア語標準表記、ローカル定型クエリを
  自然文に織り込み＝keyword-map §5：`cara download video TikTok` / `tidak bisa di-download` /
  `tanpa watermark` / `kualitas HD`。stuffing はしない）:
  - s1: 最頻原因＝誤ったリンク形式 or 私的アカウント。`tiktok.com/@namapengguna/video/ID` または
    `vt.tiktok.com` 短縮リンク・アプリ内 Bagikan からコピー・私的/友達限定はどの外部ツールでも取得不可
  - s2: 保存失敗/途中停止は TikTok CDN の一時エラー/瞬間的制限＝数分待って再貼り付け・cache 削除・
    末尾 `?...` パラメータ除去で成功率向上
  - s3: 元ファイルを取得＝**MP4 tanpa watermark（TikTok ロゴなし）** を HD 最高画質で・公開動画のみ・
    元が低解像度なら高画質化不可・標準 MP4 で iPhone/Android/PC を変換なしで再生/編集可
  - title `Video TikTok Tidak Bisa Di-download — Solusi Lengkap` / cta `Coba Pengunduh TikTok`
- section 見出し三項に `id` 分岐を追加（s1=`Periksa format URL` / s2=`Hapus cache lalu coba lagi` /
  s3=`Hanya video publik`）＝英語フォールバック "Check the URL format" 等を解消。
- 新規 URL なし（既存 id view の本文充足）＝sitemap 不変。hreflang は既存方針どおり en/ja/pt/ar+x-default
  （id は `?locale=` クエリ形式のまま。path 化は keyword-map §6 のとおり将来の別タスク）。
