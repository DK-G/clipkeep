## 実装（commit `1288df0`）

被引用性の律速は「他サイト/ツール/Google Dataset Search が**引用・harvest しやすい形**でデータが出ているか」。
既存 `/platform-status`（合成プローブ表・badge・Dataset JSON-LD）を、機械可読な一次データ配信として完成させた。

1. **公開 JSON 配信** `app/api/v1/platform-status/route.ts`（新規）
   - `buildStatusExport(snapshot, history)` を JSON 配信＝Dataset の `distribution.contentUrl`。
   - ヘッダ: `content-type: application/json`・`access-control-allow-origin: *`（オープンデータ）・`cache-control: public, max-age=300, s-maxage=300`。
   - ページと同じ KV キャッシュ snapshot を返す（母数非依存・traffic ゼロでも成立）。D1/bindings 変更なし（読み取り＋既存 KV のみ）。
2. **Dataset JSON-LD を harvest 適格へ強化**（`src/lib/platform-status/export.ts` 新規・単一ビルダー）
   - `distribution`（`DataDownload`/`application/json`/JSON URL）、`variableMeasured` を**文字列→`PropertyValue`**（value=status・uptime% を付記）、
     `temporalCoverage`（uptime 履歴の初回サンプル→checkedAt、履歴無しなら省略）、`measurementTechnique`、`keywords`（7語）を追加。
   - `buildStatusExport` / `buildDatasetJsonLd` の**単一ビルダー**で page / JSON 配信 / JSON-LD の乖離を構造的に排除。
3. **ページに raw JSON 引用リンク**を追加（Embed セクション、CC BY 4.0 明記）。
4. **テスト** `src/lib/platform-status/export.test.ts`（6件）: uptime 射影・履歴無し null 安全・temporalCoverage レンジ/省略・JSON-LD distribution/PropertyValue/temporalCoverage 省略。
