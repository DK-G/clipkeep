## 実装内容

P0-2 で骨組みだけ作った `/trend/[slug]`（全件 noindex・sitemap 未収録）に、設計 §5.3 の
**品質ゲート**を入れて「ゲートを通った live トピックのみを公開（index＋sitemap＋内部リンク）」する
機構を実装した。トピック数が現状ほぼゼロのため**今は何も公開されない**のが正しい挙動。

- `src/lib/trends/topic-store.ts`:
  - ゲート定数 `MIN_CLIPS = 3` / `MAX_LIVE_TOPICS = 12`（§9-1 の「10〜20 の下限から保守開始」）。
  - `dedupedClipCount(jobs)`: 再投稿（同一 `sourceUrl`）を畳んだ実クリップ数。
  - `listLiveTopics(kv)`: index を読み、各トピックの dedup 後クリップ数 ≥ MIN_CLIPS のものだけを
    `lastTrendedAt` 降順で `MAX_LIVE_TOPICS` 件に制限して返す。`LIVE_SCAN_CAP=200` で KV read 暴走を防止。
    sitemap 収録・内部リンク・index 可否の**単一正本**。鮮度減衰（STALE_AFTER）は P0-4 で本関数に追加予定。
- `src/lib/trends/live.ts`（新規）: `getCloudflareContext` 依存を隔離。`loadLiveTopics` /
  `loadLiveTopicsForPlatform` / `isSlugLive` を**空フォールバック**（KV 不在・ビルド時静的生成・read 失敗 → `[]`）で提供。
- `app/sitemap.ts`: `force-dynamic` 化（KV はランタイム write のため）。`loadLiveTopics()` の結果だけを
  `/trend/<slug>`（全 canonical locale, priority 0.6, daily, lastModified=lastTrendedAt）で**条件付き収録**。
  KV miss は no-op（ゲート未通過/減衰トピックは出さない）。
- `app/trend/[slug]/page.tsx`: `isSlugLive(slug)` で **index 可否をゲートに連動**（live=`robots` 既定で index、
  未達=`noindex,follow` 維持）。sitemap 収録と index を同一ゲートで同期させ「sitemap にあるが noindex」の矛盾を防ぐ。
- `app/trending/page.tsx` / `app/trending/[platform]/page.tsx`: live トピックへの**実 `<a href>` 内部リンク nav**
  （`src/components/trend-topics-nav.tsx`、孤立防止＝2026-06-17 教訓）を SSR 追加。live が無ければ何も描画しない。
  `[platform]` は KV をランタイム読みするため **`generateStaticParams` 撤去＋`force-dynamic`** で SSG（●）→
  動的（ƒ）化（ギャラリーは元々クライアント fetch のため SSR シェルのコスト増は軽微）。
