## 本番 populated-render が未検証の理由（DoD: 実行できない手順は明示）
1. **本番 KV にトピックがまだ存在しない**。当日 16:00Z（21:00Z 実行分）含め cron の `discoverTrends()` が
   `discovered={twitter:0, tiktok:0}`＝トレンド未取得（`meta:last_run` で確認: topicsWritten=0）。
   実トピックは cron がトレンドを拾った回にのみ生成される（データ依存）。
2. 検証用に本番 `TREND_KV` の `topics:index`（cron 所有の共有状態）へ一時シードを書こうとしたが、
   **セーフティ分類器が本番共有状態の上書きとして拒否**（妥当なガードレール）。本番 KV は無変更のまま
   （`topic:*` / `topics:index` とも書き込まれていないことを確認済み）。
3. ローカル miniflare KV へのシードは、この環境の **wrangler CLI バグ**（`kv key get/put` が一過性クラッシュ）で
   不成立。
- 以上より populated-render は **typecheck / build / 既存 solution ページ手本との同型性**で担保し、
  ライブの「トピック存在時レンダリング」は**実トピックが本番 KV に生成された時点**または **P0-3（sitemap 収録時に
  自然に被検証）** で確認する。これは push/deploy/gate が成功した完了タスクであり、**task-blocked ではない**
  （検証完全性の留保のみ）。
