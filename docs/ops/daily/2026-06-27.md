## 実装（ver `1c84eac0-b047-47de-8dab-a566a31ae22c`）

**問題**: 本番 worker は custom domain `clipkeep.net` に加え既定の `*.workers.dev`
（`clipkeep-web.liminality-3110.workers.dev`）でも全サイトの**完全な複製**を 200 で配信していた。
着手前計測では workers.dev の `<link rel="canonical">` は既に `https://clipkeep.net` を指していた
（成功指標の文面上はクリア）が、**クロスドメイン canonical は Google にとって「ヒント」に過ぎず無視され得る**
ため、workers.dev 側がインデックスされる duplicate-content リスクは未解消だった。

**対策**: `middleware.ts` に canonical-host 強制を追加。リクエスト `Host` が canonical ホスト
（`NEXT_PUBLIC_SITE_URL` から導出）と不一致かつ `*.workers.dev` の場合、ページ要求を
canonical origin へ **301 リダイレクト**（path + query 保持）。

- canonical ホストはデプロイ自身の `NEXT_PUBLIC_SITE_URL`（ビルド時インライン）から算出。
  本番 = `clipkeep.net`、**test デプロイ = 自身の `clipkeep-web-test.*.workers.dev`** のため
  test 環境では Host===canonical で**リダイレクトされない**（test を壊さない）。
- matcher は既存どおり `api` / `_next` / `favicon` / `manifest.json` / `robots.txt` /
  `sitemap.xml` を除外 → リダイレクト対象は HTML ページのみ（＝インデックス対象の複製のみ）。
  API は workers.dev 直叩きでデバッグ可能なまま。

typecheck / lint / build PASS（Middleware バンドル 32.6 kB）。commit `6226cc2` → `git push origin main`。
`npm run deploy:prod` 成功（Version ID 上記）。

## 本番ライブ検証（変更箇所そのもの）

`.NET HttpClient`（AllowAutoRedirect=false）で実測:

| URL | 結果 |
|---|---|
| `https://clipkeep-web.liminality-3110.workers.dev/` | **301 → https://clipkeep.net/** |
| `…workers.dev/download-reddit-video?locale=en` | **301 → https://clipkeep.net/download-reddit-video?locale=en**（query 保持） |
| `…workers.dev/solution/x-video-downloader-not-working` | **301 → https://clipkeep.net/solution/x-video-downloader-not-working**（path 保持） |
| `…workers.dev/ja` | **301 → https://clipkeep.net/ja**（locale path 保持） |
| `https://clipkeep.net/`（canonical） | **200**（リダイレクトなし・回帰なし） |
| `https://clipkeep.net/download-reddit-video?locale=en` | **200** |
| `…workers.dev/api/v1/health`（matcher 除外） | **200**（直叩き可・想定どおり非リダイレクト） |
| `…workers.dev/sitemap.xml`（matcher 除外） | **200**（clipkeep.net URL を列挙） |

注: `/ja/`（末尾スラッシュ）は両ホストで先に Next の 308（→`/ja`）が発火し、その後 workers.dev 側は
301 で clipkeep.net へ畳む（2 ホップ・許容）。

→ 成功指標「workers.dev 経由でも canonical=clipkeep.net」を**上回る**形で達成
（canonical ヒント依存ではなく 301 で複製そのものを消去）。OPS-1 の workers.dev 重複項目を消し込み。
