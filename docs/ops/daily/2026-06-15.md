## 診断（タスクの本質的ブロッカーの特定）
過去ログ（06-13/06-14）は本タスクを「GA4 認証復旧待ち」でブロック扱いにしていたが、実調査の結果ブロッカーは認証ではなかった:

1. **GA4 にカスタムディメンションが未登録**: `ad_script_load` は `ad_zone` をイベントパラメータとして送っているが（`src/components/analytics/ad-diagnostics.tsx`）、GA4 property に `ad_zone` のイベントスコープ・カスタムディメンションが未登録。`runReport` で `customEvent:ad_zone` を叩くと `400 INVALID_ARGUMENT: Field customEvent:ad_zone is not a valid dimension`（metadata の custom event dims も空）。
2. **OAuth スコープが read-only**: 現行トークンは `analytics.readonly` + `webmasters.readonly`。カスタムディメンション登録は GA4 Admin API の write 権限が必要で、自律実行では付与・登録不可（かつ過去データは遡及取得不可）。
3. **そもそも北極星が growth:review に未表示**: `growth-summary.mjs` は funnel/CVR のみ出力し、`ad_script_load` を一切表示していなかった。28日合計は `latest-ga4-events.csv` に存在（=18）するのに可視化されていなかった。

→ カスタムディメンション無しでも zone別を取得できる方式（イベント名に zone を内包する companion event）に切り替え、北極星表示を新設する方針で自律完遂。
