## 実装内容

律速は「計測ブロック」（GA4/GSC OAuth リフレッシュトークン失効 `invalid_grant`）。今日の確認でも
`fetch-ga4-report.mjs` が `invalid_grant`（"Token has been expired or revoked."）→ SA フォールバック
不在で**不透明な `ENOENT ... ga4-service-account.json` で exit 1**（真因が埋もれる）を再現。
これを「分類された WARN ＋機械可読マーカー」に変えた。

### 変更ファイル

- **新規 `scripts/lib/analytics-auth.mjs`**: 共有の認証失敗分類・WARN・ステータス出力。
  - `classifyAuthError`（`token_revoked` / `token_incomplete` / `no_credentials`）、
    `buildAuthError`（OAuth 失敗を優先分類して `AnalyticsAuthError` を投げる）、
    `printAuthWarn`（`[analytics][WARN]` バナー＝grep 可能）、
    `recordAuthStatus`（`docs/analytics/auth-status.json` に per-scope マージ書込、`blocked` 再計算）、
    `readAuthStatus`、`hasServiceAccount`。
- **`scripts/fetch-ga4-report.mjs` / `fetch-gsc-report.mjs` / `fetch-gsc-index-coverage.mjs`**:
  `getAccessToken` を OAuth→SA 全滅時に `buildAuthError` を throw する形へ。成功時は
  `recordAuthStatus(ok)`、`main().catch` で `AnalyticsAuthError` を捕捉して WARN＋ステータス記録。
  → 失効時に**真因（token revoked）と復旧手順**が必ず表に出る。
- **`scripts/run-growth-review.mjs`**: GA4 が auth ブロックなら（GSC/URL Inspection は同一資格情報なので）
  残りの fetch をスキップし、**キャッシュ済みデータでサマリを描画**、最後に統合バナー
  （`MEASUREMENT BLOCKED` ＋原因＋手順）を出して **exit 1**。非 auth エラーは従来どおり rethrow。
- **`scripts/growth-summary.mjs`**: 冒頭に **🔐 ANALYTICS AUTH** ブロックを追加。`auth-status.json` が
  `blocked` なら「数値はキャッシュ・fresh ではない」と明示＋復旧手順を表示。
- **新規 `docs/ops/analytics-auth-recovery.md`**: 失効ランブック ＋ **サービスアカウント化の調査結果・準備手順**。
  調査結論: GA4 Data API / Search Console API / URL Inspection API は **3 系統とも SA 対応**
  （GA4 プロパティ 528376605 に SA email を閲覧者、GSC `sc-domain:clipkeep.net` にユーザー追加、
  JSON キーを `.secrets/ga4-service-account.json` へ）。コード側 SA フォールバックは既存
  （`getServiceAccountAccessToken`）のため、**キー配置だけで無人運用が自動的に有効化**される。

## 計測ブロック自体の扱い（重要）

今日のタスクは「**失効を検知し WARN を出す仕組み**」と「**SA 化の準備**」であり、これは完遂済み。
ただし**計測データの fresh 化そのものは未復旧**（OAuth 再ログインまたは SA キー配置は要ユーザー手動）。
→ 計測の status は引き続き **blocked**（OAuth 失効）。GA4/GSC の実データは次のいずれかの後に取得可能:
1. ユーザーが `npm run analytics:ga4:login`（要ブラウザ）、または
2. ユーザーが `.secrets/ga4-service-account.json` を配置（手順は `docs/ops/analytics-auth-recovery.md`）。

撤退基準は測定不能週＝カウンタ非加算で非該当（判定保留・継続）。
