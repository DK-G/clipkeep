## 2. 発見促進（柱1）— locale download ページの内部リンクを path 形式へ統一

### 診断（実データ）

URL Inspection の未発見32件はほぼ全て `/download-{platform}-video` の **locale 接頭辞付き**（`/ja/` `/pt/` `/ar/`）。原因を特定:

- **内部リンクと sitemap/canonical の URL 形式が不一致**。sitemap・canonical は path 形式（`/ja/download-twitter-video`）だが、内部リンクは `?locale=` クエリ形式（`/download-twitter-video?locale=ja`）で生成されていた。
- 結果、Google にインデックスを求めている path 形式 URL には**クローラーが辿れる `<a href>` が皆無**＝sitemap 経由でしか発見されず "unknown to Google" に滞留。英語版（クエリなし path = 同一）は homepage から直リンクされ大半 indexed という非対称がこれを裏付け。

### 修正（2デプロイ）

正本ヘルパー `getLocalizedPath(path, locale)`（ja/pt/ar→path 形式、en→素 path、他 locale→`?locale=` のまま）に統一。

1. **ホーム platform グリッド** `app/page.tsx`（ver `bd876607`）: 12 プラットフォームの href をクエリ形式から `getLocalizedPath` へ。
2. **サイドメニュー（全ページ SSR の hamburger）** `src/components/side-menu.tsx`（ver `eb7e1c02`）: download 10 項目を `getLocalizedPath` へ。**全 locale ページが全 locale download ページへ path 形式でリンクする内部リンクハブ**になり、最も波及が大きい。言語スイッチャー（`?locale=` で locale ビュー切替）・utility リンクは意図的に不変更。

各デプロイで typecheck/lint/build PASS、リリースゲート **PASS=29/0/1**・D1 remote PASS。

### 本番ライブ検証

- ホーム: `/ja`・`/pt` で platform グリッドが `/ja|pt/download-*-video`（path 形式）を出力、`/`（en）は素 path のまま（回帰なし）。
- 非ホーム locale ページ（サイドメニュー経由）: `/ja/download-reddit-video`・`/ja/trending`・`/pt/download-telegram-video` で **path 形式 download リンク 10 本**を確認（`/ja/trending` は query 形式 0）。

→ 32 件の未発見 locale download ページに、最高権威のホーム＋全ページのサイドメニューから**クローラー追跡可能な path 形式 `<a href>`** が付いた。発見→クロール→index 昇格は数日〜2週間 lag、次回週次で「unknown to Google」比率の減少と新パス indexed/impression 増を監視。

### 既知の残課題（次候補・本日は対象外）

- サイドメニューの download は telegram/instagram を欠く（ホームグリッドには有り）。`/download-*-video` ページ本文の関連リンクに `?locale=` 残（各ページ3本程度）。サイドメニューの trending/latest/about 等も `?locale=` のまま。いずれも同ヘルパーで path 化可能だが、本日の測定済みクラスタ（download ページ）に絞って attribution を明確化。
