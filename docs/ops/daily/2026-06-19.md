## 根本原因（なぜ canonical が不整合だったか）
対象10ファイルの `generateMetadata` が各々インラインで
```
const url = `${base}${path}${locale !== 'en' ? `?locale=${locale}` : ''}`;
alternates: { canonical: url, languages: { en, ja, ar, es, pt, fr, id, hi, de, tr } }
```
を持っていた。これは①非英語 `?locale=` ビューが**自分自身を canonical 指定**（English へ畳まない）、②hreflang に**クエリパラメータ10言語**を列挙（distinct URL を持たない es/fr/de/tr/id/hi 含む＝canonical と矛盾）の二重問題。正本ヘルパー `src/lib/metadata-helper.ts` の `buildLocaleAlternates(path, locale)` は path-based の en/ja/pt/ar + x-default のみを hreflang に出し、非パスロケールの canonical は English に畳む（既に solution 等で稼働）。
