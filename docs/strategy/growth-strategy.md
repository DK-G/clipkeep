## 2. フェーズゲート

現状セッション 2/28日 → **右2項（回遊・ロード率）の最適化は出力ゼロのまま**。

- **Phase L（立ち上げ）**: indexed < 50 または impressions(28d) < 1,000 の間
  - 獲得（セッション数）に関わるタスクのみ選択可。柱3は凍結。
- **Phase G（成長）**: 両閾値クリア後
  - 柱3（回遊）とロード率改善を解禁。

詳細は `AGENTS.md` の Launch-Phase KPI Gate を参照（本文書と整合させること）。

## 3. 決定事項（2026-06-12 ユーザー判断）

1. **TikTok は extractor ＋ SEO の両輪で解禁**（2026-06-15 改訂、旧方針「SEO のみ・extractor は作らない」を撤回）。
   稼働中の fixer 経由 extractor（`src/lib/extract/tiktok.ts`、TikWM/Lovetik/kktiktok）を**存続**させ、
   「TikTok 保存できない/ロゴなし」系の問題解決・ガイドページで最大需要プールの検索流入を取りつつ、
   実際の抽出導線にも接続する。
   - TikTok の利用規約上のリスク（商用利用・自動アクセス）は**承認のうえ許容**する（ユーザー判断 2026-06-15）。
   - fixer API は外部依存で不安定なため、失敗時は degraded / status 導線で吸収する。
   - 規約リスク・fixer 安定性は週次レビューで監視し、悪化時は SEO 単独へ縮退できる状態を保つ。
2. **Monetag 一本化**。AdSense 審査は追わない。AdSense 向け運用制約
   （広告形式・コンテンツ品質基準）は戦略から除外する。
   法務ページ（Terms/Privacy/DMCA/Contact）はユーザー信頼のため維持。
3. **実行環境はローカル PC**（wrangler / GA4 認証があるため DoD を完結できる）。
4. **頻度: 日次1タスク + 週次戦略レビュー**。
5. **柱4（外部流入・被リンク）は廃止＝純粋に自然流入（オーガニック）のみで戦う**（2026-07-03 ユーザー判断）。
   ツールディレクトリ登録・outreach は「本プロジェクトが登録すべきものと思えない」ため実施しない。
6. **ループの作業は Claude が自律実行できる範囲に限定する**（2026-07-03 ユーザー判断）。
   人間の**継続手作業**（投稿・手動運用の反復）を前提とする施策は採らない。獲得は柱1＋柱2で完結させる。
   - **例外（許容）**: 「一度きりの人間による認証/認可を済ませれば、以降は Claude が自律運用できる」もの。
     例＝GA4/GSC のサービスアカウント認証、Cloudflare のプラン切替やクォータ認可など。
     初回の認可だけユーザーが行い、その後の運用・計測・デプロイは Claude が回す前提であれば採用可。
   - 対象外＝認可後も人間の反復操作が必要な施策（例: ディレクトリ登録の継続、SNS 手動投稿）。
7. **「ユーザーに危害が及ぶ可能性のある選択」以外は Claude の自律判断で全て進めてよい**（2026-07-03 ユーザー判断）。
   §5冒頭・決定事項6 の「戦略確定はユーザー承認」ゲートを緩和し、**柱の優先順位・組み替え・
   バックログ・戦略文書の更新も Claude が自律で確定・実行できる**。ただし監査可能性のため、
   実施内容は daily log / 更新メモに必ず残す（事後レビュー可能に）。
   - **なお承認ゲートを残す＝「危害」の定義（この範囲は事前確認する）**:
     ① 金銭（課金・有料プラン・支払いを伴う選択。例: Cloudflare 有料プラン引き上げ）、
     ② 法務/規約/ドメイン評価（§6 ガードレール＝クローキング/ドアウェイ/スパム、TikTok 規約リスク拡大、
        Instagram noindex 解除など、ペナルティ/法的責任でユーザーに不利益が及ぶもの）、
     ③ 不可逆な資産破壊（D1 データ削除、ドメイン/SEO の毀損、リポジトリ破壊）、
     ④ セキュリティ/プライバシー（`.secrets`/`.env` 露出、レート制限/WAF/広告ゾーンの弱体化）。
   - 上記①〜④以外（コード改修・デプロイ・戦略/柱の再編・コンテンツ・計測運用）は自律で進める。
   - §6 ガードレールは「危害の境界」の具体定義として不変（緩和対象ではない）。
8. **純オーガニック固定を見直し、ホワイトハット権威獲得＋独自データ資産へ転換**（2026-07-10 ユーザー承認）。
   2026-07-10 の調査で、立ち上がらない真因が **crawl budget でなくドメイン権威・被リンクの欠如**であること、
   templated 多言語ページの量産が 2025–26 品質更新で thin/doorway 判定を招くこと、当ニッチの権威格差が桁違い
   であることが確定した（daily 2026-07-10）。よって決定事項5/6 の「純オーガニック・自律限定」を**部分的に見直す**:
   - **Track A（柱1')**: templated stub 量産を停止し、**他サイトが引用したくなる独自データ資産**
     （プラットフォーム別 実稼働 status）へ転換。設計: `docs/strategy/linkable-asset-plan.md`。
   - **Track B（柱4' 再開）**: **ホワイトハット限定**の被リンク獲得（デジタルPR・関連ディレクトリ・
     コミュニティでの正当な言及）を解禁。設計: `docs/strategy/authority-plan.md`。
   - **危害ゲート①〜④・§6 ガードレールは不変**（PBN/有料リンク網/スパムは引き続き禁止。有料PRは①要承認）。

## 4. 戦略の4本柱（獲得の内訳）

### 柱1: 多言語ロングテール SEO（主軸・複利型）
- 対象: 「{platform} 動画 保存できない / 見れない / ロゴなし」系の問題解決クエリ
  × プラットフォーム（X/Reddit/Telegram/Threads/TikTok）× 言語（ja/pt/ar 優先、次に id/hi/tr）
- Monetag は tier-2/3 市場でも収益化できるため、英語ビッグワードで大手と戦わない。
- 成果物: Solution ページの内容充足、新規ロングテールページ、内部リンク。

### 柱2: トレンド鮮度ページ（速攻型）
- 既存の trending cron（毎時）を活用し、話題のバイラル動画・イベントに紐づく
  検索クエリ向けページを自動生成・更新する仕組みを作る。
- 新規ドメインでも「鮮度 × 低競合の窓」はインデックスされやすい。
- 成果物: トレンドページ生成パイプライン、トレンド × locale ページ。

### 柱3: 回遊（ページ/セッション）— Phase G まで凍結
- related / gallery / 履歴導線の強化。

### 柱1': linkable asset（独自データ資産）— **新設（2026-07-10）主軸**
- templated な not-working ページの量産を停止し、**引用・被リンクを集める独自データ資産**へ転換。
- 第一弾: プラットフォーム別「Download Status（実稼働/uptime）」データページ（既存 `extractor_jobs`＋
  合成プローブで生成、traffic 非依存）。設計: `docs/strategy/linkable-asset-plan.md`。

### 柱4': 外部流入・被リンク — **再開・ホワイトハット限定（2026-07-10 ユーザー承認）**
- 2026-07-03 の「柱4廃止・純オーガニック」を見直し、**正当な被リンク獲得を解禁**（真因＝権威に直接効く）。
- 許容: デジタルPR（独自データの寄稿/発表）、関連性の高い無料ディレクトリ、コミュニティでの正当な言及、
  埋め込みバッジによる自然被リンク。設計・実行チェックリスト: `docs/strategy/authority-plan.md`。
- **禁止（不変）**: PBN・有料リンク網・リンク交換・スパム的大量登録（§6）。有料PRは危害ゲート①＝要承認。
- `docs/ops/outreach/` の旧下書きはアーカイブのまま（新方針は authority-plan.md が正本）。

## 5. 日次タスク選択アルゴリズム

1. 計測・本番が壊れていたら最優先で修復（データ無しでの改善は禁止）。
2. `task.md` の戦略バックログ（週次レビューが優先順位を維持）の先頭から1件。
3. バックログが空なら柱1のロングテールページを1本追加。
4. **1日1タスク**。完了定義は `AGENTS.md` の Definition of Done
   （push → deploy:prod → check:release:prod → 本番確認 → daily log）。

## 6. ガードレール（禁止事項）

- クローキング・ドアウェイページ・自動生成スパムの大量投下など、
  Google ペナルティ級の手法（ドメインごと失うため）。
- 既存 fixer 経由 TikTok extractor は承認済み（2026-06-15）。ただし規約リスクを増やす拡張
  （公式 API の無断利用拡大・大規模スクレイピング化など）の追加は週次承認なしには禁止。
- Instagram の noindex / maintenance 解除。
- レート制限・広告ゾーン構成の変更（ユーザー承認が必要）。
- 本文書の書き換え（提案は週次レビュー文書に記載）。
- 法務ページ・問い合わせ導線を壊すこと。

## 7. 撤退基準

**8週連続**で GSC impressions(28d) が成長しない場合、ループは改修を停止し、
「ニッチ転換・戦略変更の提案」を週次レビューに記載してユーザー判断を仰ぐ。
